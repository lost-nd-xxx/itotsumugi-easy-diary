# ひとこま — 開発コンテキスト（Claude Code 引き継ぎ用）

このファイルは、`index.html`（単一ファイル PWA）を v2 に拡張するための仕様と設計判断をまとめたもの。
実装は Claude Code で行う。まず現状を把握し、「v2 で実装する変更」を上から順に進める。

---

## 1. プロダクト概要
一日のかけらをアイコンで簡易記録する日記 PWA。Android Chrome で「ホーム画面に追加」して使う。
- 完全無料・広告なし・**外部送信なし**（データは端末内のみ）
- 1日に何度でも記録、過去日にも記録可
- 記録パレット（アイコン＋ラベル）はユーザーが編集可能

## 2. 技術スタック / 制約（変更しない前提）
- **単一 `index.html`**（HTML/CSS/JS 全部入り）＋ `manifest.webmanifest` ＋ `sw.js` ＋ アイコン PNG。
- 保存は **IndexedDB**。`navigator.storage.persist()` で消えにくくする。
- **ローカルのみ**（Drive 等は使わない）。ただし保存処理は `Store` オブジェクトに集約し、将来差し替え可能にしてある。**この抽象化は維持すること。**
- Android の Web では `showSaveFilePicker` 等が使えないため、バックアップは **JSON 書き出し／読み込み（手動）** で行う設計。これは前提として動かさない。
- PWA 機能（インストール・永続化・SW）は **HTTPS か localhost が必須**。`file://` 不可。
- 依存ライブラリなし（バニラ JS）。この方針を維持。

> **なぜ単一 `index.html`：** ビルド工程ゼロ・依存ゼロで「腐らない」ことが目的。(1) 静的ファイルを置くだけで配信できる、(2) フレームワーク陳腐化の影響を受けず長寿命、(3) トランスパイル無しで debug が素直（行番号がソースと一致）、(4) 全体を一望でき Claude Code にも丸ごと渡せる。オフライン化は単一である必要はない（SW は複数ファイルでも可）。トレードオフは肥大化で見通しが落ちること。**次の分割はバンドラではなく、まずビルド不要のネイティブ ES モジュール**（`<script type="module">` ＋ `import` で `store.js` 等へ）。minify やコンポーネント化が要るようになって初めて Vite 等を検討する。

## 3. デザイン言語（新画面もこれに揃える）
- コンセプト：「一日の小さな瞬間が連なる**糸**」。記録は縦の糸（タイムライン）に点として連なる。アプリアイコンも糸＋3点。
- 配色は `index.html` 冒頭の CSS 変数。`:root`（ライト）と `[data-theme="dark"]`（ダーク）。`--accent` はモス green `#5c7c6e`。
- 日付見出しのみ**明朝**（`--serif`）、本文は**ゴシック**（`--sans`）。
- ライト/ダークは `theme = auto|light|dark`、`auto` は `prefers-color-scheme` 追従。
- モーダルは下からせり上がる sheet。トースト、`prefers-reduced-motion` 対応済み。新規 UI も同じ部品・トークンを使う。

## 4. 現状（v1）のデータモデルと Store API
```js
// entry (v1): { id, ts(number ms), emo, lbl, note }
// palette item (v1): { emo, lbl }
// meta: palette / theme / reminderDays / lastBackup
```
Store のメソッド（このインターフェースは維持）:
`init / addEntry / putEntry / deleteEntry / entriesInRange(startMs,endMs) / allEntries / clearEntries / getMeta(k,fb) / setMeta(k,v)`

## 5. v2 で実装する変更

### 5-1. データモデルの拡張（最重要・先にやる）
1件の記録が**複数アイコン**を持てるようにする。
```js
// entry (v2): { id, ts(number ms), icons:[ {emo, lbl, genre} ], note }
// palette item (v2): { id, emo, lbl, genre }   // genre: "mood" | "health" | "life"
```
- `genre` はジャンルキー。表示名は 気分=mood / 体調=health / 生活=life。
- 記録時、選んだパレット項目の `{emo,lbl,genre}` を **スナップショットで** `icons[]` に格納（後でパレットを変えても過去記録は不変）。
- **ジャンルはデータ駆動にする**：固定ハードコードではなく `meta.genres = [{key,label}, ...]`（既定 = mood/health/life の3件）を持ち、パレット表示・絞り込み・設定アコーディオンはこの配列を元に**動的描画**する。**ただしジャンルの増減 UI（CRUD・並べ替え・削除時整合）は v2 スコープ外**。今は固定3ジャンルで運用し、将来 UI を足すだけで開放できる状態にしておく（履歴は `icons[]` に genre をスナップショットで持つので壊れない）。

**マイグレーション**（起動時に1回、`meta.schema` で冪等化）:
- v1 entry（`emo/lbl` があり `icons` 無し）→ `icons:[{emo,lbl,genre:null}]` に変換、`emo/lbl` は削除。
- v1 palette item（`genre` 無し）→ `genre:"life"` を暫定付与（ユーザーが後で再分類可）。
- 完了後 `meta.schema = 2` を保存。
- IndexedDB のストア構成（`entries` keyPath:id / index:by_ts、`meta`）は変えない。インデックス追加が必要なら `DB_VER` を上げる。

### 5-2. 記録の操作フロー（select → commit に一本化）
- パレットのアイコンタップで**選択/解除をトグル**（複数可、ジャンルをまたいでOK）。選択中は見た目で明示。
- 下部に目立つ**「記録」ボタン**（親指で届く位置）。押すと選択中アイコンすべてで**1件**を作成し、選択をクリア。
- **即記録は廃止**（タップ＝選択）。単一でも「選択＋記録ボタン」。
- パレットの下に**折りたたみ「詳細設定」**（拡縮）。中身は **日時（datetime-local, 既定=現在）** と **メモ**。既定は閉じ。開いて値を入れれば、その日時・メモで記録（過去日時の細かい指定もここで）。記録後はリセット。
- カレンダーで日付をタップして「その日に追加」した場合は、記録画面の日時既定を**その日**にする（時刻は正午など。詳細設定で調整可）。
- 過去記録は「カレンダーから日付指定で追加」＋「詳細設定で日時を直接指定」の**両方**を用意（ユーザー希望）。

### 5-3. ホーム＝「紡いだ糸」（全日付横断フィード）
- 日付に依存せず、**全記録を新しい順**に表示。各項目に **日付＋時刻＋アイコン（複数なら並べる）＋メモ**。
- 日付が変わる位置に**淡い日付ディバイダ**を挿入。
- 初期 **10件**、「**もっと見る**」で +10 ずつ。
- ホームの日付送り ‹ › は**廃止**（日付アクセスはカレンダー経由）。
- 見出し名は「紡いだ糸」。
- 既存の「きょうの糸」単日タイムラインは、カレンダーで日をタップしたときの**その日ビュー**として残す（追記・編集の場）。

### 5-4. カレンダー
- 月表示・月送り ‹ ›。
- **週の始まりは設定のトグルに従う（既定=日曜、月曜も選べる）**。`meta.weekStart = 0|1`（0=日曜, 1=月曜）。曜日ヘッダと先頭オフセットの描画だけ切り替える（データ・履歴には一切影響しない）。
- 各セルに**その日の最後の記録のアイコン**。最後の記録が複数アイコンなら横に小さく数個（最大3程度）。
- 日をタップ → その日の糸（5-3 の単日ビュー）を開き、追記・編集できる。

### 5-5. 絞り込み
- **ジャンル（気分/体調/生活）＋個別アイコン**を複数選択。
- 判定は **OR**：1件のうち**どれかのアイコンが該当すれば表示**。
- 反映先は **ホームのフィードとカレンダーの両方**。カレンダーは非該当日を淡く（該当日のセルは「最後の**該当**記録」のアイコン）。

### 5-6. 下部バー / 設定の移動
- 下部バーを **［カレンダー｜ホーム｜絞り込み］** に置換（**ホームを中央**）。
- v1 の「今日／書き出し」は廃止。**設定は右上ハンバーガー ☰** に集約。**書き出しは設定内へ移動**。

### 5-7. 記録パレットのジャンル表示（記録画面）
- パレットを **3 ジャンルのセクション**に分け、各セクションを **＋ / − で展開・折りたたみ**。
- 折りたたみ状態は **IndexedDB に保存**（`meta.collapse = {mood:bool, health:bool, life:bool}`）して、毎回の操作を軽減。
- **この折りたたみ状態は JSON 書き出しに含めない**（端末固有の UI 状態）。初期は全展開。

### 5-8. 設定（ハンバーガー内）
- テーマ（auto/light/dark）。
- **週の始まり**：日曜／月曜のトグル（既定=日曜、`meta.weekStart`）。
- **記録パレット**を **ジャンルごとに折りたたみ可能なアコーディオン**に。各項目は 絵文字／ラベル／並べ替え（↑↓、ジャンル内）／削除、項目追加時に**ジャンルを選択**。※**ジャンル自体の追加・削除・改名 UI は v2 では作らない**（5-1 のとおりデータ駆動だけ用意）。
- データ：書き出し（JSON）／読み込み／バックアップ催促間隔／永続化依頼。
- About。

### 5-9. 既定パレット（各ジャンル6個。横6列に揃う）
絵文字は提案。ユーザーが容易に変更できる前提。
- **気分(mood)**：楽しい😄／穏やか🙂／普通😐／眠い😴／ややつらい😟／つらい😣
- **体調(health)**：頭痛🤕／腹痛😖／生理🩸／服薬💊／疲労😮‍💨／眠気😪
- **生活(life)**：制作✍️／遊び🎮／休息☕／お出かけ🚶／運動🏃／買い物🛒

### 5-10. 書き出し / 読み込み（v2）
- 書き出し JSON：`version:2`、`palette`（genre 付き）、`genres`（`meta.genres`）、`entries`（icons[]）、`settings`（`theme` / `reminderDays` / `weekStart`）。**collapse 状態は含めない**（端末固有の UI 状態）。
- 読み込み：**v1・v2 両対応**。v1 entry は読み込み時に `icons[]` へ変換。マージ（id 重複スキップ）／全置換は現行の確認ダイアログ方式を踏襲。

## 6. 表示まわりの細目（決定済み）
- 複数アイコンの1件：糸ではアイコンを横に並べて表示、ラベルは「・」連結。
- カレンダーのセル：最後の記録（絞り込み時は最後の該当記録）のアイコン。複数なら最大3個小さく。
- 絞り込みOR判定（1件内のいずれかが該当で表示）。

## 8. 運用メモ
- アプリ更新時は `sw.js` の `CACHE = "hitokoma-vN"` の版番号を上げる（旧キャッシュ破棄）。
- アイコン差し替えは `icon-*.png` を同名で。
- 配信は HTTPS（自サイト FTP か GitHub Pages）。
- 著作権：発想（アイコン日記）は保護対象外。素材・コードは自作のものを使う（他アプリの素材/コード/固有デザインの流用はしない）。
